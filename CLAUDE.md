## 작업 원칙

1. 사용자가 만든 변경사항을 임의로 되돌리지 않는다.
2. 요청 범위를 벗어난 리팩터링을 수행하지 않는다.
3. 기존 코드 스타일을 유지한다.
4. DECISIONS.md의 내용을 준수한다.
5. 코드 수정 전 아래 절차를 따른다.

### 코드 수정 절차

1. 현재 상태 분석
2. 수정 계획 제시
3. 사용자 승인
4. 구현

단, 사용자가 명시적으로 수정 또는 구현을 요청한 경우 즉시 작업할 수 있다.
